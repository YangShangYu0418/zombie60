# DESIGN — Zombie60

## 1) 遊戲定位
- 類型：喪屍末日經營 + 事件敘事（小說感）
- 節奏：高壓、短回合決策、資源緊縮
- 情境：**封閉空間的長期生活**（避難所/據點內「活下去」與「相處下去」同等重要）
- 目標：每次選擇都會留下「資源代價」或「關係代價」

## 2) 主要資源（可先固定 6 個）
- 食物 Food（數量）
- 飲水 Water（數量）
- 醫療 Med
- 彈藥 Ammo
- 士氣 Morale
- 噪音 Noise（越高越容易引發攻擊/事件）
- 探索風險 ExploreRisk（外出/探索次數與行為會累積，讓威脅壓力遞增）

### 飲食品質（收束到資源管理的關鍵）
- 另外追蹤 **DietQuality（飲食品質）**：由「你今天怎麼分配食物/水」決定
- 飲食品質會長期影響角色的 **Body（體態）** 與狀態：
  - 低品質/缺乏營養 → 逐步變瘦、體力下降、易生病（但消耗較少）
  - 高品質/營養均衡 → 逐步更健康、曲線更豐滿、恢復更快（但消耗更多/更稀缺）
- 重要：體態變化是「長期趨勢」而非一次事件劇變（更像經營遊戲）

## 3) 角色屬性（原型）
每人 0–5：
- 体力 STR（外出/負重/耐受）
- 理性 LOG（修理/規劃/風險控）
- 社交 CHA（交易/說服/緩和衝突）
- 戰鬥 COM（遭遇戰/守家）

### 角色長期狀態（影響敘事與管理）
- Health（健康）
- Stress（壓力）
- Infection（感染風險）
- Body（體態）：例如 0=消瘦 → 100=豐滿（用於敘事描寫與部分事件/好感加成）

## 4) 事件卡格式（草案）

### 固定週期檢定（非人威脅主軸）
- 每日或每 N 日觸發一次「喪屍壓力檢定」（非人為主）：
  - 由 Noise、ExploreRisk、據點防禦、防守人選（COM）共同決定
  - 檢定結果會造成：資源損失、受傷/感染、設備損壞、或迫使你做妥協決策
- 原則：探索越多 → ExploreRisk 越高 → 壓力遞增、事件更兇

## 4) 事件卡格式（草案）
事件以 JSON 定義：
- id, title, dayMin/dayMax, tags
- text（敘事段落）
- choices[]：每個選項包含
  - label（選項文字）
  - cost（資源消耗）
  - check（可選：要求某角色/屬性）
  - outcomes（成功/失敗/分支）
  - effects（狀態變化）
  - hooks（任務線推進/旗標）

## 5) 任務線（小說式章節）
- 任務由「旗標 flags」推進（例如：foundRadio, metNeighbor, rescuedKid 等）
- 每條任務線拆成 3–7 個節點（章節）
- 章節由事件卡觸發，並可回寫 flags

## 6) 關係/好感度（多結局核心）
- 角色：男主角 + 3 位女主（各有特色、能力互補）
- 每位女主各自有 **Affection（好感度）** 與 **Trust（信任）**（-100~+100 或 0~100）
- 事件選項會同時影響：
  - 物資/健康/噪音/士氣（生存線）
  - Affection/Trust（關係線）
- 長期封閉生活的壓力會累積：
  - 「互相依賴」→ 好感/信任更容易上升
  - 「情緒爆炸」→ 一次事件可造成關係崩塌、甚至死亡或分裂

## 7) 結局框架（可共存的多路線）
- **後宮結局**：三人好感/信任均達標 + 關鍵旗標達成
- **特定女主結局**：某一人達最高門檻，其他人低於排他門檻
- **單人結局**：其他人離開/死亡/關係破裂，玩家獨活
- **死光結局**：資源崩壞、感染、內鬥或攻擊造成全滅
- 結局判定應結合：day、flags、存活名單、好感/信任、重大事件選擇
- 額外可加：Body/DietQuality 作為「結局氛圍差異」與特定事件門檻（不搶主系統戲份）

## 8) MVP 成功標準
- 60 秒搜刮可玩（移動+撿取+倒數）
- 日結算可玩（消耗+事件+選擇+狀態）
- 至少 20 張事件卡、1 條任務線能完整跑完
- 可以存檔/讀檔
